# UX設計

claude-code-usage-barのユーザーインターフェース・インタラクション設計をまとめる。

---

## メニューバー表示

### 通常状態: "CC 5h 42% / 7d 18%"

メニューバーには常に最新の使用率を簡潔に表示する。

| 要素 | 表示例 | 説明 |
|------|--------|------|
| プレフィックス | `CC` | Claude Codeの略称。アプリの識別子 |
| 5時間枠 | `5h 42%` | `rate_limits.five_hour.used_percentage` |
| 区切り | `/` | 5時間枠と7日枠の区切り |
| 7日枠 | `7d 18%` | `rate_limits.seven_day.used_percentage` |

**フォーマット**: `CC 5h {n}% / 7d {n}%`

表示例:
- `CC 5h 42% / 7d 18%` — 通常
- `CC 5h 0% / 7d 3%` — 低使用率
- `CC 5h 95% / 7d 78%` — 高使用率

### 警告状態: 色変化・アイコン変化

使用率に応じてメニューバーの表示を変化させ、視覚的に状態を伝える。

| 使用率 | 状態 | アイコン | 色 |
|--------|------|---------|-----|
| 0-69% | 通常 | `gauge.low` | デフォルト（システムカラー） |
| 70-84% | 注意 | `gauge.medium` | オレンジ |
| 85-94% | 警告 | `gauge.high` | 赤 |
| 95-100% | 危険 | `exclamationmark.triangle` | 赤（点滅なし、静的表示） |

SF Symbolsを使用し、macOSの標準的なメニューバーアイコンスタイルに準拠する。

### コンパクト表示オプション

メニューバースペースが限られるユーザー向けに、以下の表示モードを設定で切り替え可能とする。

| モード | 表示例 | 説明 |
|--------|--------|------|
| 標準 | `CC 5h 42% / 7d 18%` | 5時間枠と7日枠の両方を表示 |
| 数値のみ | `CC 42 / 18` | パーセント記号を省略、最もコンパクト |
| 5時間枠のみ | `CC 5h 42%` | 日常的に最も重要な指標のみ |
| アイコンのみ | （アイコン） | テキストなし、色で状態を伝達 |

デフォルトは「標準」。コンパクト表示でもポップオーバーで全情報を確認可能。

## ポップオーバー

メニューバーアイテムをクリックすると、`MenuBarExtra(.window)`スタイルのポップオーバーが表示される。

### 使用率ゲージ（5h / 7d）

5時間枠と7日枠それぞれの使用率を視覚的なゲージで表示する。

```
┌──────────────────────────────────┐
│  Claude Code Usage               │
│                                  │
│  5h limit      42%   reset in 2h 13m  │
│  ████████░░░░░░░░░░░░            │
│                                  │
│  7d limit      18%   reset Tue 09:00  │
│  ████░░░░░░░░░░░░░░░░            │
│                                  │
│  Context       31%               │
│  Model         Sonnet / Opus     │
│  Session       project-name      │
│  Last update   14 sec ago        │
└──────────────────────────────────┘
```

| 要素 | 説明 |
|------|------|
| ゲージバー | `SwiftUI ProgressView`または`Gauge`で実装 |
| パーセンテージ | `used_percentage`の値を整数表示 |
| 色 | メニューバーと同じ閾値ベースの色変化（通常/注意/警告/危険） |

### リセット時刻カウントダウン

`resets_at`フィールドから現在時刻との差分を計算し、相対的な残り時間を表示する。

| 残り時間 | 表示形式 |
|---------|---------|
| 1時間以上 | `N時間M分後` |
| 1時間未満 | `M分後` |
| 1分未満 | `まもなく` |
| 過去の時刻 | `リセット済み（更新待ち）` |

### コンテキスト使用率

`context_window`の情報を表示する。

```
┌─────────────────────────────┐
│  コンテキスト                 │
│  ██████░░░░░░░░░░░░░  32%   │
│  45,000 / 200,000 tokens    │
└─────────────────────────────┘
```

### セッション一覧

複数のClaude Codeセッションが同時に稼働している場合、各セッションの状態を一覧表示する。

| 列 | 内容 |
|-----|------|
| セッションID | `session_id`の先頭8文字 |
| モデル | `model`フィールド |
| コスト | `cost.total_cost`の推定値（USD） |
| 最終更新 | データの最終受信時刻からの経過時間 |

アクティブセッションが1つの場合はセッション一覧を省略し、ゲージ表示のみとする。

### 設定へのアクセス

ポップオーバー下部に設定ボタンを配置。

| 設定項目 | 説明 |
|---------|------|
| 表示モード | 標準 / 5時間枠のみ / アイコンのみ |
| 通知閾値 | 70% / 85% / 95% の有効/無効 |
| ログイン時に起動 | SMAppServiceの有効/無効切り替え |
| collectorの状態 | インストール済み / 未設定 / エラー |

## データ欠落時の表現

### 「値なし」（rate_limits未受信）

セッションは存在するが`rate_limits`フィールドがまだ受信されていない状態。最初のAPI応答を受信するまでこの状態になる。

| 表示箇所 | 表現 |
|---------|------|
| メニューバー | `CC —` |
| ポップオーバー | ゲージをグレーアウト + `Last data: N min ago` + 「Open Claude Code or run a prompt to refresh usage.」メッセージ |

### 「未使用」（セッション未開始）

collectorがインストールされているが、Claude Codeセッションがまだ開始されていない状態。

| 表示箇所 | 表現 |
|---------|------|
| メニューバー | `CC —` |
| ポップオーバー | 「No active session. Start Claude Code to begin tracking.」メッセージ |

### 「古いデータ」（最終更新からN分経過）

最終データ受信から一定時間が経過した状態。セッション終了やClaude Code停止を示唆する。

| 経過時間 | メニューバー | ポップオーバー |
|---------|------------|-------------|
| 5分以上 | テキストを薄い色（`secondary`）に変更 | `Last update: N min ago` |
| 30分以上 | `CC —` | `Last data: N min ago` + 「Session may have ended.」 |
| 2時間以上 | `CC —` | 「No recent data. Session likely ended.」 |

### 「非対応プラン」（API/Team等）

`rate_limits`フィールドが提供されないプラン（API、Team）のユーザー向け表示。

| 表示箇所 | 表現 |
|---------|------|
| メニューバー | `CC —` |
| ポップオーバー | 「Rate limits are available on Pro/Max plans.」メッセージ + `context_window`と`cost`のみ表示 |

`rate_limits`がなくても`context_window`や`cost`は表示可能なため、これらの情報はポップオーバーに表示する。

## 通知

### 使用率閾値通知（70/85/95%）

使用率が閾値を超えた際にmacOS通知を送信する。

| 閾値 | 通知タイトル | 通知本文 |
|------|------------|---------|
| 70% | Claude Code 使用率 | 5時間枠の使用率が70%に達しました |
| 85% | Claude Code 使用率（警告） | 5時間枠の使用率が85%に達しました。ペースを調整してください |
| 95% | Claude Code 使用率（危険） | 5時間枠の使用率が95%に達しました。まもなく制限されます |

通知の制御:

- 各閾値につき1回のみ通知（同一リセット期間内での重複防止）
- リセット後に閾値カウンターをリセット
- 5時間枠と7日枠それぞれ独立に通知
- ユーザーが設定で各閾値の通知を個別に無効化可能

### リセット時刻通知

リセット時刻の到来を事前に通知する（オプション機能）。

| タイミング | 通知内容 |
|-----------|---------|
| リセット15分前 | 「5時間枠があと15分でリセットされます」 |
| リセット時 | 「5時間枠がリセットされました」 |

## アクセシビリティ

### VoiceOver対応

すべてのUI要素にアクセシビリティラベルを設定する。

| 要素 | アクセシビリティラベル |
|------|---------------------|
| メニューバーアイテム | 「Claude Code使用率。5時間枠42パーセント、7日枠18パーセント」 |
| 使用率ゲージ | 「5時間枠使用率ゲージ、62パーセント」 |
| リセットカウントダウン | 「リセットまで2時間34分」 |
| 設定ボタン | 「claude-code-usage-bar設定を開く」 |

```swift
Text("CC 5h 42% / 7d 18%")
    .accessibilityLabel("Claude Code使用率。5時間枠42パーセント、7日枠18パーセント")
```

### Dynamic Type

ポップオーバー内のテキストはDynamic Typeに対応し、ユーザーのシステム設定に従ってフォントサイズを自動調整する。SwiftUIの標準フォントスタイル（`.body`, `.caption`, `.headline`）を使用することで自動的に対応される。

メニューバーのテキストはシステムフォントサイズに固定（macOSメニューバーの標準動作）。
